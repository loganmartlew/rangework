## Overview Screen Review

### Friction issues

1
Welcome card does too much
For returning users this card is pure noise — they already know the app. The explainer paragraph and email address waste space on every single visit. Consider collapsing to just the action buttons after first use, or remove the body copy entirely.
2
Stat cards aren't tappable
Two full-height cards just to display "1" each is expensive. In M3, overview summary tiles should navigate — tapping "1 session" should go to Sessions. Side-by-side metric cards with a tap affordance would halve the height and eliminate a navigation step.
3
"Next move" is static and not actionable
The advice never changes and has no button — it tells you what to do but doesn't help you do it. Either make it contextual (driven by the user's actual data) or replace it with something actionable, like a shortcut into the first incomplete unit.
4
No quick-access to recent content
A returning user wanting to run last week's session has to go to Sessions, find the template, and open it. A "recently used" row here — just one or two cards — would make the Overview genuinely useful on return visits, not just at setup.

### Summarised Findings

**The two biggest problems** are the welcome card and the missing recency. The welcome card is optimised for a first-time user but shown to every returning user on every visit — the long explainer paragraph and email address burn vertical space that could show something actually useful. And right now the Overview has no memory of what the user has done: no last session, no recently edited unit, nothing. That forces an extra navigation step every time someone returns to continue their work.

**The stat cards** could earn their space by being tappable — in M3, a summary tile on an overview screen is naturally a navigation shortcut. Right now they're decorative. Making them tappable (with a subtle chevron) would mean a returning user can tap "1 session" and land directly in Sessions without touching the nav bar.

**"Next move"** has the right instinct — nudging the user toward productive behaviour — but it's static text with no action attached. It would be much stronger as a contextual, data-driven prompt: something like "Your '50-yard pitches' unit has no ball counts — add some" with a button that takes you straight there.

On the M3 side: the button pair implicitly ranks sessions above units (filled vs. outlined), which may or may not be intentional. And the app bar saying "Rangework Overview" alongside the logo is redundant — the screen title alone is enough.
